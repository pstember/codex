# Project Brain

This directory stores durable project memory for future agents and collaborators.

## Rules

- Add dated entries in `YYYY-MM-DD` format.
- Record why a decision was made, not only what was decided.
- Update stale entries instead of duplicating conflicting information.
- Keep entries concise and useful for future work.
- Do not store secrets, credentials, personal tokens, or transient logs.

## Files

- `decisions.md`: product, architecture, and implementation decisions.
- `learnings.md`: discoveries, constraints, gotchas, and lessons.
- `open-questions.md`: unresolved product or technical questions.
